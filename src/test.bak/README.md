# API接口测试文档

## 文件说明

### auth.http
包含二手交易平台用户认证相关接口的完整测试用例：
- 用户注册接口测试（正常注册、各种异常情况）
- 用户登录接口测试（正常登录、各种异常情况）
- 用户状态检查接口测试
- 登录后功能测试（获取用户信息、检查登录状态）
- 用户登出测试

### http-client.env.json
HTTP客户端环境配置文件，定义不同环境的基础URL：
- `dev`: 开发环境 (http://localhost:8080)
- `test`: 测试环境 (http://localhost:8080)
- `prod`: 生产环境 (https://api.secondhand.com)

## 使用方法

### 1. 在IntelliJ IDEA中使用
- 直接打开 `auth.http` 文件
- 点击每个请求旁边的绿色运行按钮执行测试
- 可以通过右上角的环境选择器切换不同的测试环境

### 2. 测试顺序建议
1. 首先运行注册接口测试，创建测试用户
2. 运行登录接口测试，获取认证token
3. 使用获取的token测试需要认证的接口
4. 最后测试登出功能

### 3. 注意事项
- 确保后端服务已经启动并监听在对应端口
- 数据库连接正常
- Redis服务正常运行（用于Sa-Token会话管理）
- 第一次运行时可能需要先注册测试用户
