## 推荐使用方式

建议优先使用 `auth-simple-test.http` 进行日常测试，因为它语法简单且无IDE警告。

如果需要测试复杂场景（如动态变量引用、并发登录等），可以使用 `auth-api-test.http`。

### 1. 环境配置
确保 `http-client.env.json` 文件配置正确：
```json
{
  "dev": {
    "baseUrl": "http://localhost:8080"
  }
}
```

### 2. 启动服务
确保后端服务已在本地8080端口启动：
```bash
mvn spring-boot:run
```

### 3. 运行测试
在IntelliJ IDEA中：
1. 打开对应的 `.http` 文件
2. 点击每个测试用例左侧的运行按钮 ▶️
3. 或者使用 Alt+Enter 快捷键运行整个文件

💡 小贴士：
- 可以右键点击文件选择 "Run All Requests" 执行全部测试
- 测试结果会在底部的HTTP Client控制台显示
- 绿色对勾表示成功，红色叉号表示失败

### 4. 查看结果
- 成功响应：绿色对勾 ✓
- 失败响应：红色叉号 ✗
- 可以在底部的HTTP Client控制台查看详细响应

## 测试数据说明

### 默认测试账户
- 用户名：`admin`
- 密码：`123456`

### 测试注意事项
1. **数据库准备**：确保MySQL数据库中有初始的admin用户
2. **Redis服务**：部分功能依赖Redis缓存
3. **测试顺序**：建议按文件中的顺序执行测试
4. **清理测试数据**：测试产生的用户数据需要手动清理

## 常见问题

### Q: 测试返回404错误
A: 检查服务是否在8080端口正常运行

### Q: 登录测试失败
A: 确认数据库中存在用户名为"admin"且密码为"123456"的用户

### Q: 注册测试失败
A: 检查数据库连接和Redis服务状态

### Q: Token相关测试失败
A: 确认Sa-Token配置正确，检查application.yml中的相关配置
