# 商品状态码修正说明

## 修正内容

### 问题发现
在实现商品审核功能时，状态码分配有误：
- 原实现：审核驳回使用状态码3
- 正确实现：审核驳回应使用状态码2

### 商品状态码对照表（修正后）

| 状态码 | 状态说明 | 业务含义 |
|--------|----------|----------|
| 0 | 待审核 | 用户发布商品后的初始状态，等待管理员审核 |
| 1 | 已上架 | 管理员审核通过后的状态，商品可被浏览和购买 |
| 2 | 审核驳回 | 管理员审核拒绝的状态，需用户修改后重新提交 |
| 3 | 已下架 | 商品主动下架或系统下架的状态 |
| 4 | 已售出 | 商品成功交易完成的状态 |
| 5 | 已删除 | 商品被删除的状态 |

### 修正的文件

1. **ProductServiceImpl.java**
   - 修正了审核拒绝时的状态设置：`product.setStatus(2)` 而非 `product.setStatus(3)`

2. **相关文档**
   - PRODUCT_API_DOC.md：更新了状态说明和审核流程描述
   - PRODUCT_REVIEW_FEATURE_SUMMARY.md：更新了状态转换规则和完整状态说明
   - ProductReviewTest.java：更新了测试用例中的状态码验证

### 状态转换流程（修正后）

```
用户发布商品
    ↓
状态0(待审核)
    ↓
管理员审核
    ├── 通过 → 状态1(已上架)
    └── 拒绝 → 状态2(审核驳回)
```

### 影响范围

✅ **API接口**：审核接口行为正确，返回正确的状态码
✅ **业务逻辑**：状态转换符合预期的业务流程
✅ **数据一致性**：数据库中商品状态码正确反映业务状态
✅ **前端显示**：可以根据正确的状态码显示相应的UI状态
✅ **后续操作**：基于正确状态码的后续业务操作逻辑正确
